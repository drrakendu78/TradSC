# StarTrad FR v4.1.6

## Editeur de bindings — refonte capture & mapping

- **Capture native DirectInput** via `gilrs` : tes vJoy, HOTAS et sticks DirectInput (invisibles au `navigator.getGamepads` du browser) sont maintenant lus directement. Support jusqu'à 128 boutons par device.
- **Mapping dynamique js&lt;N&gt;** comme dans le jeu : le premier stick que tu presses devient js1, le deuxième différent js2, etc. Persisté par profil pour rester stable entre sessions.
- **Cards** : sticks physiques par défaut sur `startrad_user_bindings_LIVE.xml`, vJoy déclarées visibles uniquement sur les profils importés.
- **Masquage par hardware** (œil sur la carte) : masquer MOZA / Keychron / HID Bluetooth resserre la numérotation pour que Sol-R [R] devienne joystick:2 directement.
- **Bluetooth HID host** : filtré automatiquement de la liste des joysticks.

## Fixes

- Persistence des cartes masquées et du dropdown "Lié à" corrigée (race condition useEffect qui écrasait les données après F5).
- Boutons en check digital seul (plus de flicker sur les boutons analog au repos type triggers/grips).
- Bindings orphelins (`js<N>` référençant un slot inexistant dans le profil) affichés "Non attribué" au lieu d'afficher "Joystick N".
- Filtre des doublons hardware quand Windows enumère un même stick via USB et HID.

## Détails techniques

- Module Rust `joystick_input.rs` (`gilrs 0.11`) avec deux commandes Tauri : `read_active_joystick_inputs` (polling boutons + axes) et `list_gilrs_joystick_devices` (énumération).
- Resolver de capture : mapping `gilrsSlot → cardId` persisté en localStorage par profil.
